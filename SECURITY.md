# 安全策略（Security Policy）

感谢你帮助改进 `MolWorld` 的安全性。

本项目包含上传、审核、管理控制台、文件移动/删除、SQLite 数据读写等能力。若你发现与这些能力相关的安全问题，请按本文说明反馈。

## 支持范围

当前优先处理以下类型问题（包括但不限于）：

- 管理员认证/会话绕过
- 越权访问管理 API
- 文件上传/文件删除相关安全问题（路径穿越、任意文件读写等）
- SQL 注入 / 命令注入 / XSS / CSRF
- 敏感信息泄漏（管理员凭据、环境变量、日志泄漏等）
- 限流绕过、批量操作滥用导致的高风险问题

## 不建议公开披露的内容

在问题修复前，请不要在公开 Issue 中直接发布：

- 可直接利用的 payload
- 管理员接口的复现细节（特别是带凭据、cookie 的请求）
- 服务器路径、数据库内容、环境变量值
- 可复现破坏性操作的完整步骤

## 报告方式

优先方式（推荐）：

1. 使用 GitHub 的私密漏洞报告功能（`Security` / `Report a vulnerability`，若仓库已启用）。

如果仓库暂未启用私密报告：

1. 先提交一个**不含利用细节**的公开 Issue（仅说明“存在安全问题，需要私下沟通”）。
2. 等待维护者提供私下沟通渠道后，再补充完整复现信息。

## 建议提供的信息

为便于快速定位，请尽量包含：

- 问题类型与影响范围（例如：未授权访问 `/api/admin/...`）
- 复现步骤（脱敏后）
- 预期行为 vs 实际行为
- 影响版本 / 分支（如 `main`、某个 commit）
- 环境信息（本地/生产、浏览器、是否使用 Cloudflare/反代）
- 截图、响应状态码、日志片段（请脱敏）

## 响应与修复（目标）

这是个人维护项目，处理速度可能受时间影响，但会尽量按以下节奏处理：

- 72 小时内确认收到
- 7 天内完成初步判断（是否确认、影响范围、临时缓解建议）
- 修复后再公开说明（必要时在 README / 文档补充排查与修复记录）

## 使用与部署安全建议（自部署）

如果你自行部署本项目，建议至少做到：

- 使用 HTTPS（生产环境）
- 为管理员设置强密码，避免默认值
- 不提交 `.env` / `.env.production`
- 持久化并保护 `prisma/`、`public/uploads/`、`public/memes/`
- 反代/CDN（如 Cloudflare）开启后，验证上传与图片链路是否正常
- 定期更新依赖（Next.js / Prisma / sharp 等）

## 说明

- 本项目是开源应用模板/站点项目，不提供任何安全保证或 SLA。
- 负责任披露将被优先处理，并在修复后同步文档。
